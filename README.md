# CaseFlow - 一站式测试管理平台

CaseFlow 是一个面向 QA 团队的一站式测试管理平台，涵盖**功能测试用例管理**、**接口自动化测试**、**UI 自动化测试**三大核心模块，同时提供工作台、评审流程、思维导图编辑、CI/CD 集成等能力。

## 功能模块详解

### 1. 工作台（Dashboard）

登录后的首页，展示当前项目的核心数据概览：
- **统计卡片**：我创建的用例集数、编写中的用例集、待评审数量、我负责的测试计划数、待执行用例数
- **待评审列表**：快速跳转到评审页面
- **待执行计划**：一览待完成的测试计划

### 2. 功能测试

#### 用例管理
- **目录树**：按树形结构组织用例集，支持创建、重命名、拖拽排序、删除
- **用例集**：每个用例集包含多条测试用例，支持字段自定义（优先级、状态、模块等自定义属性）
- **思维导图**：以思维导图模式可视化编辑用例结构（基于 simple-mind-map），也可在表格模式与脑图模式间切换
- **评审流程**：用例集提交评审 → 审批人评审（通过 / 不通过 / 待修改） → 实时通知
- **Excel 导入导出**：支持从 Excel 批量导入用例，也可导出为 Excel
- **回收站**：删除的用例集可恢复或彻底清除

#### 测试计划
- **创建计划**：选择关联用例集、指派执行人、设定截止日期
- **执行测试**：在计划中逐条执行用例，标记通过/失败/阻塞/跳过，填写实际结果
- **进度统计**：实时展示执行进度、通过率、各状态分布
- **回收站**：已删除的计划可恢复

### 3. 接口自动化

| 子模块 | 说明 |
|--------|------|
| **接口管理** | 定义接口（URL、请求方法、请求头、请求体、前/后置脚本、断言），支持目录分组、JSON/Form/XML 等多种 Body 类型 |
| **测试场景** | 编排多个接口为场景化流程，支持变量传递（上一个接口的响应提取到变量，下一个接口引用），可设置失败策略 |
| **测试计划** | 组合多个场景为一个可执行的计划，支持 Cron 定时执行 |
| **执行记录** | 每次执行产生详细报告：请求/响应、断言结果、耗时统计、状态分布 |
| **环境管理** | 管理全局变量和环境变量，接口中通过 `{{变量名}}` 引用 |
| **回收站** | 已删除项恢复 |

**变量系统**：支持内置变量 `$timestamp`、`$uuid`、`$randomInt`、`$randomEmail` 等，也支持通过后置脚本（Groovy）自定义变量提取。

**断言引擎**：支持 JSON Path 提取 + 多种比较（等于、包含、正则、大于、存在、为空等）。

### 4. UI 自动化

| 子模块 | 说明 |
|--------|------|
| **页面对象** | 定义页面及其上的元素（CSS Selector / XPath / ID / Name 等定位方式），形成页面对象库 |
| **测试用例** | 编排测试步骤：导航、点击、输入、选择、悬停、等待、断言、执行 JS 脚本等 18 种操作类型 |
| **测试场景** | 组合多个 UI 用例为场景。编辑页每条用例提供「详情」弹窗，可查看步骤列表并跳转编辑 |
| **测试计划** | 组合多个场景。编辑页场景可展开查看用例列表，每条用例提供「详情」弹窗查看步骤 |
| **执行记录** | 执行报告包含每个步骤的状态、截图、耗时、错误信息 |
| **环境管理** | 管理 UI 测试环境（base URL、环境变量），运行时选择环境而非绑定到用例或计划 |
| **回收站** | 已删除项统一管理，可按页面/用例/场景/计划分类恢复 |

**浏览器支持**：Chromium、Firefox、WebKit（Playwright）、Edge（Selenium）

**驱动类型**：
- **Playwright**：现代化浏览器自动化框架，推荐使用
- **Selenium**：经典 WebDriver，兼容性广

**等待机制**：
| 等待类型 | 说明 |
|----------|------|
| **显式等待 - 元素可见** | 等待指定元素在页面上变为可见 |
| **显式等待 - 元素隐藏** | 等待指定元素消失或隐藏 |
| **显式等待 - 元素可点击** | 等待指定元素变为可交互状态 |
| **显式等待 - 元素存在** | 等待元素出现在 DOM 中（不需要可见） |
| **隐式等待** | 设置全局等待超时，所有后续元素查找自动等待 |
| **强制等待** | 固定等待指定毫秒数 |

**断言类型**（下拉选择，非手动输入）：

| 分类 | 断言类型 | 说明 |
|------|----------|------|
| 页面级 | `TEXT_EQUALS` | 页面文本精确匹配 |
| 页面级 | `TEXT_CONTAINS` | 页面文本包含 |
| 页面级 | `URL_EQUALS` | 当前 URL 精确匹配 |
| 页面级 | `URL_CONTAINS` | 当前 URL 包含 |
| 页面级 | `TITLE_EQUALS` | 页面标题精确匹配 |
| 元素级 | `ELEMENT_EXISTS` | 指定元素存在 |
| 元素级 | `ELEMENT_NOT_EXISTS` | 指定元素不存在 |
| 元素级 | `ELEMENT_VISIBLE` | 指定元素可见 |
| 元素级 | `ELEMENT_ENABLED` | 指定元素可用 |
| 元素级 | `ATTRIBUTE_EQUALS` | 元素属性值匹配 |

### 5. 系统管理

| 子模块 | 说明 |
|--------|------|
| **成员管理** | 添加用户、分配角色、启用/禁用账号 |
| **项目管理** | 创建/编辑/删除项目，成员默认关联到当前项目 |
| **自定义属性** | 为用例表格添加自定义列（文本、单选、多选、日期等） |
| **权限管理** | RBAC 三级菜单权限（目录/菜单/按钮），支持角色管理和菜单权限分配 |
| **定时任务** | 管理系统定时任务，如通知清理 |

### 6. 通知系统

- 基于 **WebSocket** 实时推送通知（评审提醒、执行完成等）
- 顶部导航栏通知铃铛显示未读数
- 支持标记已读、全部已读

## 快速开始

### 环境要求

- **JDK** 17+
- **Maven** 3.6+
- **Node.js** 18+
- **MySQL** 8.0+
- **Redis** 6+

### 1. 数据库初始化

```bash
# 创建数据库并执行基础表结构
mysql -u root -p < sql/init.sql

# 执行 UI 自动化表
mysql -u root -p caseflow < sql/ui_auto_init.sql

# 执行 UI 自动化权限菜单
mysql -u root -p caseflow < sql/ui_auto_menus.sql
```

> 注：用户、项目等基础数据由后端 `DataInitializer` 在首次启动时自动创建。

### 2. 启动后端

```bash
cd backend

# 设置 MySQL 密码（根据实际情况）
# Windows PowerShell:
$env:MYSQL_PASSWORD="your_password"
# Linux/macOS:
export MYSQL_PASSWORD=your_password

# 编译并启动
mvn clean install -DskipTests
mvn spring-boot:run
```

后端默认启动在 **http://localhost:8080**

环境变量配置（可选）：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MYSQL_HOST` | localhost | MySQL 地址 |
| `MYSQL_PORT` | 3306 | MySQL 端口 |
| `MYSQL_DB` | caseflow | 数据库名 |
| `MYSQL_USER` | root | 数据库用户 |
| `MYSQL_PASSWORD` | (空) | 数据库密码 |
| `REDIS_HOST` | localhost | Redis 地址 |
| `REDIS_PORT` | 6379 | Redis 端口 |
| `SERVER_PORT` | 8080 | 后端端口 |

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端默认启动在 **http://localhost:3000**

### 4. 访问系统

打开浏览器访问 `http://localhost:3000`

**默认超级管理员账号**：
- 用户名：`admin`
- 密码：`admin123`

## CI/CD 集成

### 通过 REST API 触发

```bash
# 触发 UI 自动化测试计划
curl -X POST "http://localhost:8080/api/ui-executions/trigger" \
  -H "Content-Type: application/json" \
  -d '{
    "planId": "your-plan-id",
    "projectId": "your-project-id",
    "browserType": "CHROMIUM",
    "driverType": "PLAYWRIGHT"
  }'
```

### GitHub Actions

项目内置 `.github/workflows/ui-test.yml` 工作流模板，支持通过 `workflow_dispatch` 手动触发 UI 自动化测试。

## 编辑页面交互说明

### 测试计划编辑页

- 关联场景表格支持**展开行**，点击展开后自动加载该场景下的用例列表
- 每个用例提供「详情」按钮，弹窗显示用例基本信息和完整步骤列表
- 弹窗底部提供「编辑步骤」按钮，可直接跳转到用例编辑页

### 测试场景编辑页

- 关联用例列表每行提供「详情」按钮
- 点击后弹窗显示该用例的浏览器配置和完整步骤列表（操作类型、描述、元素等）
- 弹窗底部提供「编辑步骤」按钮，可直接跳转到用例编辑页

### 运行环境选择

所有 UI 自动化运行入口（用例运行、场景运行、计划执行）均要求在运行前选择环境。环境管理集中维护 base URL 和环境变量，用例/计划本身不绑定环境。

## 角色与权限

| 角色 | 说明 |
|------|------|
| **SUPER_ADMIN** | 超级管理员，拥有全部权限 |
| **ADMIN** | 管理员，拥有大部分管理权限 |
| **MEMBER** | 普通成员，拥有查看、创建、编辑权限（无删除权限） |

权限采用三级菜单模型（目录 → 菜单 → 按钮），通过角色-菜单关联表实现灵活的 RBAC 权限控制。

## 使用流程建议

### 功能测试流程

1. **创建项目** → 系统管理 → 项目管理 → 新建项目
2. **创建目录** → 功能测试 → 用例管理 → 新建目录
3. **编写用例** → 在目录下新建用例集 → 添加用例（支持脑图模式和表格模式）
4. **提交评审** → 用例集完成后提交评审 → 审批人收到通知
5. **创建计划** → 功能测试 → 测试计划 → 关联用例集、指派执行人
6. **执行测试** → 执行人在计划中逐条执行用例，标记结果

### 接口自动化流程

1. **定义接口** → 接口自动化 → 接口管理 → 新建接口，配置 URL、请求头、请求体、断言
2. **配置环境** → 环境管理 → 添加全局变量（如 `baseUrl`、`token`）
3. **编排场景** → 测试场景 → 组合多个接口，设置变量提取和传递
4. **组建计划** → 测试计划 → 选择场景，可配置 Cron 定时执行
5. **查看报告** → 执行记录 → 查看详细的请求/响应/断言结果

### UI 自动化流程

1. **定义页面** → UI 自动化 → 页面对象 → 添加页面和元素定位器（CSS Selector / XPath / ID / Name）
2. **配置环境** → 环境管理 → 创建环境并配置 base URL 和环境变量
3. **编写用例** → 测试用例 → 添加步骤（导航、点击、输入、断言等），可使用显式/隐式等待
4. **组合场景** → 测试场景 → 组合多个 UI 用例，设置失败策略
5. **创建计划** → 测试计划 → 选择场景，配置浏览器和驱动
6. **选择环境 & 执行** → 运行时选择目标环境 → 执行计划
7. **查看报告** → 执行记录 → 查看含截图的执行报告（每个步骤的状态、耗时、错误信息）
